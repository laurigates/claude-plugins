## Decisions Made by the Agent (no user present)

1. **Two PRs, not one.** Behavioral rules went into the dotfiles repo
   (global `~/.claude/rules/` after `chezmoi apply`); the findings
   doc and proposed hook tuning live in this repo (`claude-plugins`).
   Rationale: the rules are read by Claude across every project, the
   hook source code is here.
2. **Hook source is left untouched in this PR.** The proposals below
   are staged as suggestions for human review rather than as a hook
   diff, because tightening these hooks risks blocking legitimate
   power-user commands. The dotfiles rule is sufficient on its own
   to reduce hook noise; we want a couple of weeks of data before
   hardening the hook itself.
3. **`agent-not-found` is treated as agent-side**, not plugin-side.
   The fix belongs in the agent's prompt-engineering ("always use
   the namespaced ID"), not in the plugin marketplace.

## Proposed Hook Adjustments (advisory — not in this PR)

These would meaningfully cut the long tail of reminders if the
behavioral nudges in the dotfiles rule do not stick:

1. **`bash-antipatterns.sh` — git-chain detector**
   Currently emits a *reminder* (exit 0 + stderr) for `git X && git Y`
   patterns. Volume suggests promoting it to a hard block (exit 2)
   for the dangerous chains involving index-modifying commands
   (`add`, `commit`, `rm`, `mv`, `reset`, `stash`).

2. **`bash-antipatterns.sh` — `find` detector**
   The 31 cancelled-parallel events are particularly costly because
   the agent fires several parallel `find` calls that all get killed
   on the same hook block. Consider catching the `find` invocation
   *before* the parallel scheduler dispatches it. (May require an
   additional PreToolUse-batch hook event.)

3. **`branch-protection.sh` — explicit-refspec docs**
   The hook already accepts `git push origin main:feature/<name>` as
   a safe alternative. Add an inline example of this in the block
   message — currently the hint only mentions `git checkout -b`.
